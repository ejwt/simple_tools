功能：分割二进制文件。有两种工作模式：

模式0――等分分割模式
一直复制到来源文件的末尾，并生成用于合并分割后的文件块的批处理文件join.bat。若原文件名为filename1，则分割后的文件块名为filename2NNNNfilename3，filename2、filename3可以在程序运行的时候指定。

模式1――指定起始地址、结束地址分割模式，输入十进制的起始地址startAddress、结束地址endAddress，两个地址（相对于文件首偏移量）之间的数据被保存到一个新文件中。注意：“startAddress=0”表示文件首，所以切割下来的文件的大小等于(endAddress-startAddress+1)字节。

该程序的局限：由于DOS命令行字符个数的限制，当文件个数太多或文件名太长时，join.bat无效。这时可用copy /B *.extention_name joined_filename之类的命令连接所有文件，由于分割出来的文件块名很有规律，这样合并文件，在Windows 2000的DOS虚拟机下是可以的（操作系统自动按照文件名排列文件），与源文件完全相同。在Windows XP 32-bit专业版上也是可以的。注意：当文件个数很多时，要注意观察，好像有时候排列顺序会乱。原因不清楚。
――――――――――――――――――――――――――――――――――――――――
Build in Linux:

gcc split_binary_file.c -D_FILE_OFFSET_BITS=64
mv a.out split_binary_file_v5

算法说明：如何产生分割后的文件块名？原来想用字符数组，但试了一下，发现因为有变动的部分（编号），而且文件块的名字也不定，不易实现。就采用如下方法：先将文件块名产生出来，存放在一个临时的文本文件dst_block_name.tmp中，在写各个文件块时，若需要文件名，就从dst_block_name.tmp中读取。程序运行结束前自动删除这个临时文件。

v5.0对v4.0的改动：
模式1――不论文件多大，切割精度一律是字节。不必选择文件是否大于2GiB。
当文件分割的块数大于9999时，只是警告，用户可以选择继续。
――――――――――――――――――――――――――――――――――――――――
v4.0对v3.0的改动：
模式0――不论文件多大，切割精度一律是字节。每个文件块不能超过2GiB。最多可以将文件分割为9999块。
模式1――对于大于等于2GiB的文件，切割精度是MiB；对于小于2GiB的文件，切割精度是字节。
――――――――――――――――――――――――――――――――――――――――
v3.0对v2.1的改动：
增加对大于等于2GiB的文件的处理。对于大于等于2GiB的文件，切割精度是MiB；对于小于2GiB的文件，切割精度是字节。
最多可以将文件分割为8192块。
